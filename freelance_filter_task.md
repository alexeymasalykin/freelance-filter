# Задание: Telegram скрипт-фильтр фриланс заказов

## Что нужно сделать
Написать Python-скрипт на Telethon, который слушает входящие сообщения от Telegram-бота @Freelance_find_bot, фильтрует их по цене и стоп-словам, и пересылает подходящие заказы в отдельную Telegram-группу.

## Технический стек
- Python 3.11+
- Telethon (asyncio)
- Запуск локально на macOS

## Telegram API credentials
- `api_id`: 2040
- `api_hash`: "b18441a1ff607e10a989891a5462e627"
- Это официальные credentials Telegram Desktop из открытого исходного кода

## Формат входящих сообщений от бота
Бот @Freelance_find_bot присылает сообщения в личку в таком формате:

```
Создать онлайн-приглашение на свадьбу
📋 ID: 306304
🔗 Источник: Биржа
💰 Цена: 1500.0 RUB
🎯Категория: 🔑 Сайты под ключ
Нужно сделать онлайн-приглашение на свадьбу, состоящее из
- обложки
- даты, времени
- места
...
```

## Логика фильтрации

### Минимальная цена
- Пересылать только заказы с ценой >= 5000 RUB
- Если цена не указана или не распарсилась — пересылать (не терять потенциально хороший заказ)

### Стоп-слова (регистронезависимо)
Если в тексте сообщения встречается любое из этих слов — НЕ пересылать:
- 1С, Битрикс, Bitrix
- WordPress, WP
- Laravel
- Java (но не JavaScript)
- Unity
- n8n
- Make (как платформа автоматизации, но аккуратно — слово "make" может быть в обычном тексте. Фильтровать только если "Make" или "make.com")
- Zapier
- Tilda, Тильда
- Wix
- Webflow
- Bubble
- Shopify
- OpenCart
- Joomla
- Drupal
- ModX

### Стоп-слова: важные нюансы
- "Java" не должно срабатывать на "JavaScript" или "Java Script"
- "WP" — только как отдельное слово (не часть другого слова)
- Проверять и кириллицу и латиницу где применимо (1С, Битрикс/Bitrix, Тильда/Tilda)

## Куда пересылать
- В Telegram-группу (ID группы указать в конфиге)
- Пересылать как forward оригинального сообщения (чтобы можно было перейти к боту и открыть заказ)

## Конфигурация
Все настройки вынести в config.py или .env файл:
- api_id, api_hash
- Номер телефона для авторизации
- ID группы для пересылки
- Минимальная цена
- Список стоп-слов

## Авторизация
- При первом запуске Telethon попросит ввести номер телефона и код из Telegram
- Сессия сохраняется в .session файл, повторная авторизация не нужна

## Логирование
- Логировать в консоль: получено сообщение, прошло/не прошло фильтр, причина отклонения
- Формат: `[ВРЕМЯ] СТАТУС: краткое описание`

## Структура проекта
```
freelance-filter/
├── main.py          # Точка входа, запуск клиента
├── config.py        # Все настройки
├── filter.py        # Логика парсинга и фильтрации
├── requirements.txt # telethon
└── README.md        # Инструкция по запуску
```

## Инструкция по запуску (включить в README)
1. `pip install -r requirements.txt`
2. Заполнить config.py (телефон, ID группы)
3. `python main.py`
4. При первом запуске ввести код из Telegram
5. Скрипт работает в фоне и фильтрует заказы

## Дополнительно
- Скрипт должен корректно обрабатывать ошибки (бот не отвечает, нет сети, FloodWait)
- При FloodWait — ждать указанное время и продолжать
- Graceful shutdown по Ctrl+C
